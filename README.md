# Bun + Express + Drizzle + PostgreSQL Template

A robust, type-safe, and high-performance backend template utilizing modern JavaScript/TypeScript tooling.

## 🚀 Tech Stack

- **Runtime & Package Manager**: [Bun](https://bun.sh/)
- **Language**: [TypeScript](https://www.typescriptlang.org/)
- **Framework**: [Express](https://expressjs.com/)
- **Database**: [PostgreSQL](https://www.postgresql.org/)
- **ORM**: [Drizzle ORM](https://orm.drizzle.team/)
- **Validation**: [Zod](https://zod.dev/)
- **Documentation**: [TypeDoc](https://typedoc.org/)

## 📂 Project Structure

```
├── src/
│   ├── app.ts            # Express app setup and middleware configuration
│   ├── index.ts          # Application entry point
│   ├── config/           # Environment variables and configuration files
│   ├── db/               # Database connection, schemas, migrations, and seeds
│   ├── modules/          # Feature-based modules (routes, controllers, services)
│   └── shared/           # Shared utilities, middleware, constants, and types
├── docs/                 # Documentation generated by TypeDoc
├── tests/                # Test suites
├── .env                  # Environment variables
├── drizzle.config.ts     # Drizzle ORM configuration
└── package.json          # Dependencies and scripts
```

## 🛠️ Getting Started

### Prerequisites

- [Bun](https://bun.sh/) installed on your machine
- [PostgreSQL](https://www.postgresql.org/) running locally or remotely

### Installation

1. Clone this repository (or use it as a template).
2. Install dependencies:
   ```bash
   bun install
   ```
3. Set up your environment variables:
   Update the `.env` file in the root directory with your database credentials and other necessary secrets.

### Database Setup

Run database migrations and seed your database using Drizzle ORM commands (ensure your `.env` is configured properly).

## 💻 Commands

The following scripts are available in `package.json`:

- **Development**:
  ```bash
  bun run dev
  ```
  Runs the application in watch mode using Bun.

- **Production Build & Start**:
  ```bash
  bun run build
  bun run start
  ```
  Builds the TypeScript code and starts the compiled JavaScript output.

- **Type Checking**:
  ```bash
  bun run typecheck
  ```
  Runs the TypeScript compiler without emitting files.

- **Documentation**:
  ```bash
  bun run docs
  bun run docs:watch
  ```
  Generates project documentation using TypeDoc.

- **Testing**:
  ```bash
  bun run test
  bun run test:watch
  bun run test:coverage
  ```
  Runs tests using Bun's native test runner.

## 📐 Coding Standards & Guidelines

This template strictly follows the guidelines outlined in `AGENTS.md`. Key principles include:

- **General**: Use explicit types, prefer ES modules, and write self-explanatory code. Avoid using `any`.
- **Naming Conventions**: `camelCase` for variables/functions, `PascalCase` for classes/interfaces/types, `UPPER_SNAKE_CASE` for constants, and `kebab-case` for files.
- **API Design**: Follow RESTful principles. Use plural resource names and return consistent JSON responses (`{ "success": true, "data": {} }` or `{ "success": false, "message": "..." }`).
- **Error Handling**: Use centralized error middleware. Never swallow errors.
- **Validation**: Validate all incoming requests. Never trust client input.
- **Database**: Use parameterized queries through Drizzle ORM. Keep data access logic centralized.
- **Security**: Never hardcode secrets. Always read from environment variables.
- **Documentation**: Add TSDoc comments to all exported functions, classes, interfaces, and types. Keep TypeDoc documentation up to date.
